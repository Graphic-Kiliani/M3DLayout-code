# Infinigen Processing Pipeline
We provide a complete processing pipeline for the Infinigen dataset.

Before running the pipeline, please download the Infinigen subset from our [M3DLayout dataset](https://huggingface.co/datasets/Metaverse-AI-Lab/M3DLayout/tree/main/scene_dataset/infinigen) or generate Infinigen-Indoors data from the official repository:

👉 https://github.com/princeton-vl/infinigen/blob/main/docs/HelloRoom.md


including extracting layout parameters, rendering multi-view images, and generating LLM-based scene descriptions from `.blend` scene files.

## Prerequisites

| Dependency | Version | Notes |
|------------|---------|-------|
| [Blender](https://www.blender.org/download/) | 4.4+ (tested with 5.0.1) | Download the Linux portable build and place it next to the scripts |
| Python packages | `python-dotenv`, `openai` | Auto-installed into Blender's bundled Python on first run |
| OpenAI API key | — | **Optional.** Only needed for Step 6 (LLM description) |

## Quick Start

```bash
cd ./data_curation/infinigen

# Place your Blender build here (or pass its path as 2nd argument)
#    e.g.: blender-5.0.1-linux-x64/blender

bash run_pipeline_infinigen.sh <path_to_infinigen_output_seed_folder> (which is usually a timestamp seed name such as '765a3dbf')
```

That's it. All outputs go to `<seed_folder>/output/`.

## Input

A seed folder generated by Infinigen with the following structure:

```
765a3dbf/
├── coarse/
│   └── solve_state.json      # Object placement solver state
├── fine/
│   └── scene.blend            # Full scene file
└── ...
```

## Output

All outputs are written to `<seed_folder>/output/`, with filenames prefixed by the seed:

```
765a3dbf/output/
├── parameters_765a3dbf.json                        # Step 1: raw scene parameters
├── all_765a3dbf.json                               # Step 2: merged with solve_state
├── render_topdown_765a3dbf.png                     # Step 3: top-down render
├── render_topdown_annotated_765a3dbf.png           # Step 3: annotated (numbered bboxes)
├── main_objects/                                   # Step 3: multi-view object renders
│   ├── X_Sofa.png, Y_Sofa.png, ...                #   5 views × top-N objects
│   ├── X_Sofa_annotated.png, ...                   #   annotated versions
│   └── render_main_objects_765a3dbf.png            #   concatenated grid
├── render_oblique_765a3dbf_oblique_0.png           # Step 4: oblique views (up to 4)
├── render_oblique_765a3dbf_oblique_1.png
├── render_oblique_765a3dbf_oblique_2.png
├── render_oblique_765a3dbf_oblique_3.png
├── floormask_765a3dbf.png                          # Step 5: binary floor mask (1024×1024)
└── description/
    └── description_topdown.txt                     # Step 6: structured JSON description
```

## Pipeline Steps

| Step | What it does | Time* | Output |
|------|-------------|-------|--------|
| 1 | **Export scene parameters** — opens `.blend`, extracts AABB/OBB, materials, mesh stats for all objects | ~1 min | `parameters_<seed>.json` |
| 2 | **Merge with solve_state** — enriches parameters with Infinigen's object placement solver data | <1s | `all_<seed>.json` |
| 3 | **Render top-down** — orthographic overhead render + annotated version with numbered bounding boxes + multi-view renders of top-5 furniture items | ~12 min | `render_topdown_*.png` + `main_objects/` |
| 4 | **Render oblique views** — up to 4 angled perspective renders of the full room | ~1 min | `render_oblique_*_oblique_*.png` |
| 5 | **Render floor mask** — white floor silhouette on black background | <10s | `floormask_<seed>.png` |
| 6 | **LLM description** — sends annotated topdown + oblique + main objects images to GPT-4o for structured scene description | ~20s | `description/description_topdown.txt` |

*\*Approximate times on NVIDIA H20. CPU-only rendering will be significantly slower.*

## Configuration

### Blender Path

The script auto-detects Blender in three ways (in order):

1. **Second CLI argument**: `bash run_pipeline_infinigen.sh <seed> /path/to/blender`
2. **Bundled**: a `blender-*/blender` directory next to the script
3. **System PATH**: `which blender`

### OpenAI API (Step 6 — Optional)

Create a `.env` file in the repo root:

```bash
# Required
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx

# Optional: API relay/proxy for regions where OpenAI is not directly accessible
OPENAI_BASE_URL=https://your-relay.example.com/v1

# Optional: model name (default: gpt-4o)
OPENAI_MODEL=gpt-4o
```

Alternatively, set these as environment variables before running the script.

**If you are in a region where OpenAI is blocked**, you **must** set `OPENAI_BASE_URL` to a compatible relay endpoint. Without it, Step 6 will fail with a 403 error.

If no API key is configured, Step 6 is gracefully skipped and Steps 1–5 run normally.

### Network Proxy (Optional)

If your environment needs a proxy to reach PyPI (for auto-installing dependencies) or the OpenAI API, edit the `[USER CONFIG] Network proxy` section in `run_pipeline_infinigen.sh`:

```bash
export http_proxy="http://your-proxy:port"
export https_proxy="http://your-proxy:port"
```
